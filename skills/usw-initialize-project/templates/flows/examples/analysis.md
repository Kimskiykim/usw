# Пример flow: analysis

> **Ненормативный пример.** Этот файл не запускается на месте. Скопируйте его в
> `<flows.root>/<name>.md`, адаптируйте под проект и только затем запускайте
> `$usw-run-flow <name> "<task>"`.

Помогает превратить исходное намерение в ограниченное и проверяемое описание
изменения. Конкретные skills, формат артефактов и review gates выбираются под
проект и задачу.

## Пример процесса

1. Проверить входные данные и уточнить границы задачи.
2. Если намерение неоднозначно, разобрать по одному нерешённому вопросу.
3. Сравнить несколько подходов и зафиксировать выбранный.
4. Описать ожидаемое поведение, критерии готовности и затронутые области.
5. Разбить слишком широкую работу на независимо проверяемые части.
6. Показать результат владельцу и дождаться решения перед реализацией.

## Что адаптировать

- Названия и расположение planning artifacts.
- Обязательные проверки и reviewers.
- Условия остановки и вопросы, требующие решения человека.
- Разрешённые записи для Analysis в вашем проекте.
